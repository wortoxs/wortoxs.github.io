从零开始的 React 学习 - Part 2

组件

React 允许将 HTML标记、CSS 以及 Javascript 在组件中进行组合使用，以此来达到 UI 元素复用的目的。

一些 React 组件库:

  • Material UI
  • Chakra UI
  • Ant Design
  • Arco Design
  • Semi Design

定义组件

组件即为 Javascript 函数，但需要注意:

  • 需要将组件导出以供其他模块使用，例如export default
  • 组件必须以大写字母开头，否则无法使用
  • 若元素与return不位于同一行，则需要用()包含起来，否则return之下的元素会被忽略

使用组件

导入和导出组件

在项目中，通常需要将组件划分在不同的文件中，以便组件的复用和查找。

export default 和 export

Syntax Export statement Import statement
Default export default function Button() {} import Button from './button.js';
Named export function Button() {} import { Button } from './button.js';
  • default export在文件中仅能有一个，而named export在文件中能有多个
  • default的导入和导出名称可以不同, 例如上例中可以使用import Banana from './button.js', named方式必须相同

JSX

JSX 是 Javascript 的语言扩展，其允许在 js 中使用类似 HTML 的文档标记,但比 HTML 更加严格。

JSX 的规则

  1. 仅允许返回单个元素
    由于 JSX 最终被编译为 Javascript，在不适用数组等方法的情况下，函数不能返回多个值，因此若要返回多个元素，需要将其使用单个元素进行包装

  2. 闭合所有标签
    对于 self-closing tag，例如 img，则为 <img />
    对于 wrapping tag，例如 li，则为 <li>...</li>

  3. camelCase
    在 JSX 翻译为 Javascript 时，其中的属性则翻译为 Javascript 对象中的键。由于 Javascript 的变量命名规则，其键名不能出现 - 以及 Javascript 中的关键字，因此使用 camelCase 来进行命名。

  • 名称中包含-的属性，例如stroke-width则命名为strokeWidth
  • 名称中包含关键字，例如class则修改为className

在 JSX 中使用变量

使用 {}来在 JSX 中使用 Javascript 变量，其中变量可以在两个地方使用

  • JSX 标签中，例如 <p>{name}</p>
  • 用于属性值的等号后，例如 <img src={src} />

{{}}表示变量为 Javascript 对象，可以表示 CSS

props

向组件中传递 props，类似于 HTML 中的属性，若要接收传递的值，则需要在函数参数中接受，并且可以设置默认值，若参数为 undefined 时则应用。

对于变量名和属性相同的情况，可以使用 spread syntax 来简化代码

对于传递的子组件，则使用 children 属性接收。

props 是只读的，当需要改变时，由父组件向其传递一个新的 props。如果要进行交互，则需要使用 state。

Pure

组件应当是无副作用的，即相同的输入，应当有相同的输出。
